# 彩票大模型 API 接口文档

## 0. 本次规则修复说明

本次后端修复了彩票号码生成规则，尤其是大乐透号码格式。

### 后端变更

- 新增彩种规则校验，所有大模型返回的号码都会先按玩法规则检查。
- 大乐透必须返回 7 个号码：5 个前区号码 + 2 个后区号码。
- 大乐透前区范围：`01-35`，不重复。
- 大乐透后区范围：`01-12`，不重复，后区号码会带 `+` 前缀。
- 如果某个大模型返回号码不合规，会跳过该模型候选，并在 `candidates[].error` 里返回原因。
- 如果所有大模型都返回不合规，后端会使用本地规则生成兜底号码。
- `recommended_numbers` 现在会尽量保证是符合当前 `game` 玩法规则的最终号码。

### 前端需要配合

前端不要再假设所有彩票都是同一种展示规则，需要按 `game` 区分展示：

| game | 展示方式 |
|---|---|
| `ssq` | 前 6 个为红球，最后 1 个 `+xx` 为蓝球 |
| `dlt` | 前 5 个为前区，最后 2 个 `+xx` 为后区 |
| `fc3d` | 3 个数字位 |
| `pl3` | 3 个数字位 |
| `pl5` | 5 个数字位 |

大乐透示例：

```json
{
  "game": "dlt",
  "recommended_numbers": ["02", "12", "19", "25", "30", "+03", "+09"]
}
```

前端展示时应该显示成：

```text
前区：02 12 19 25 30
后区：03 09
```

不要只截取前 5 个号码。带 `+` 的号码是后区号码，展示时可以去掉 `+`，但内部保存建议保留 `+`，方便后端兑奖校验。

### 多模型候选失败处理

如果多模型场景下某个模型生成不合规，响应里会出现：

```json
{
  "provider": "model-b",
  "model": "xxx",
  "reply": "模型原始回复",
  "recommended_numbers": [],
  "error": "模型返回的号码不符合该彩票玩法规则，已跳过该候选。"
}
```

前端可以：

- 最终展示只用顶层 `recommended_numbers`
- 调试页面或日志里展示 `candidates`
- 不要把 `error` 候选当成可用号码展示给用户

## 1. 基础信息

服务地址：

```text
http://127.0.0.1:8000
```

鉴权方式：

```http
X-API-Key: 后端配置的 API_ACCESS_KEY
```

如果后端 `.env` 里的 `API_ACCESS_KEY` 为空，则不校验密钥。

跨域配置：

```env
CORS_ALLOWED_ORIGINS=*
```

默认允许所有来源跨域。正式环境如果要收紧权限，可以改成真实前端域名，多个地址用英文逗号分隔。

支持的彩票类型：

| game | 名称 |
|---|---|
| `ssq` | 双色球 |
| `dlt` | 大乐透 |
| `fc3d` | 福彩3D |
| `pl3` | 排列三 |
| `pl5` | 排列五 |

号码生成规则：

| game | 规则 | 返回格式示例 |
|---|---|---|
| `ssq` | 6 个红球，01-33，不重复；1 个蓝球，01-16 | `["01","06","11","18","22","31","+09"]` |
| `dlt` | 5 个前区，01-35，不重复；2 个后区，01-12，不重复 | `["03","12","19","25","30","+04","+07"]` |
| `fc3d` | 3 位数字，每位 0-9，可重复 | `["1","2","3"]` |
| `pl3` | 3 位数字，每位 0-9，可重复 | `["1","2","3"]` |
| `pl5` | 5 位数字，每位 0-9，可重复 | `["1","2","3","4","5"]` |

## 2. 大模型配置

### 单模型配置

只配置一个大模型时，聊天接口会直接调用该模型生成号码。

```env
LLM_BASE_URL=https://api.openai.com/v1
LLM_API_KEY=your-model-api-key
LLM_MODEL=gpt-4o-mini
LLM_PROVIDER_NAME=default
```

### 多模型配置

配置 `LLM_PROVIDERS` 后，会覆盖上面的单模型配置。聊天接口会并发调用多个大模型生成候选号码，然后由第一个模型作为评审模型，对候选结果进行讨论、选择或综合生成最终号码。

```env
LLM_PROVIDERS='[
  {
    "name": "model-a",
    "base_url": "https://api.openai.com/v1",
    "api_key": "key-a",
    "model": "gpt-4o-mini"
  },
  {
    "name": "model-b",
    "base_url": "https://example.com/v1",
    "api_key": "key-b",
    "model": "qwen-plus"
  }
]'
```

多模型返回里会包含：

- `generation_mode`: `multi_model_discussion`
- `candidates`: 每个模型的候选结果
- `decision_reason`: 最终选择或重新生成的理由

## 4. 查询今日彩票开奖

### 接口地址

```http
GET /api/v1/lottery/today
```

### 请求头

| 参数 | 必填 | 说明 |
|---|---|---|
| `X-API-Key` | 条件必填 | 后端配置了 `API_ACCESS_KEY` 时必填 |

### Query 参数

| 参数 | 类型 | 必填 | 默认值 | 说明 |
|---|---|---|---|---|
| `game` | string | 否 | `ssq` | 彩票类型 |

### 响应参数

| 字段 | 类型 | 说明 |
|---|---|---|
| `game` | string | 彩票类型 |
| `draw_date` | string | 开奖日期，格式 `YYYY-MM-DD` |
| `issue` | string/null | 期号 |
| `numbers` | array[string] | 开奖号码 |
| `source` | string | 数据来源 |
| `raw` | object/null | 原始数据 |

### 响应示例

```json
{
  "game": "ssq",
  "draw_date": "2026-04-30",
  "issue": "20260430",
  "numbers": ["01", "06", "11", "18", "22", "31", "+09"],
  "source": "local_file",
  "raw": {
    "draw_date": "2026-04-30",
    "issue": "20260430",
    "numbers": ["01", "06", "11", "18", "22", "31", "+09"]
  }
}
```

## 5. 聊天生成彩票号码

### 接口地址

```http
POST /api/v1/chat/generate
```

### 请求头

| 参数 | 必填 | 说明 |
|---|---|---|
| `Content-Type` | 是 | 固定为 `application/json` |
| `X-API-Key` | 条件必填 | 后端配置了 `API_ACCESS_KEY` 时必填 |

### 请求体

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|---|---|---|---|---|
| `session_id` | string/null | 否 | null | 会话 ID。不传则后端自动创建新会话 |
| `game` | string | 否 | `ssq` | 彩票类型 |
| `message` | string | 是 | 无 | 用户输入内容 |
| `include_today_result` | boolean | 否 | true | 是否把今日开奖数据一起提供给模型 |

### 请求示例

```json
{
  "game": "ssq",
  "message": "给我一组号码",
  "include_today_result": true
}
```

继续同一个会话时，把响应里的 `session_id` 传回来：

```json
{
  "session_id": "6505e80daea046fb88335404db8c9b4f",
  "game": "ssq",
  "message": "换一组偏小号的",
  "include_today_result": true
}
```

### 响应参数

| 字段 | 类型 | 说明 |
|---|---|---|
| `session_id` | string | 会话 ID，前端继续聊天时传回 |
| `game` | string | 彩票类型 |
| `reply` | string | 最终回复内容 |
| `recommended_numbers` | array[string] | 最终推荐号码 |
| `llm_used` | boolean | 是否实际调用了大模型 |
| `generation_mode` | string | 生成模式：`fallback`、`single_model`、`multi_model_discussion`、`multi_model_fallback` |
| `candidates` | array[object] | 多模型候选结果；单模型时也会返回一个候选 |
| `decision_reason` | string/null | 多模型讨论后的最终选择理由 |
| `today_result` | object/null | 今日开奖结果查询结果 |

### candidates 字段

| 字段 | 类型 | 说明 |
|---|---|---|
| `provider` | string | 模型配置名称 |
| `model` | string | 模型名称 |
| `reply` | string | 该模型生成的原始回复 |
| `recommended_numbers` | array[string] | 从该模型回复中提取的号码 |
| `error` | string/null | 该模型调用失败时的错误信息 |

### 单模型响应示例

```json
{
  "session_id": "6505e80daea046fb88335404db8c9b4f",
  "game": "ssq",
  "reply": "推荐号码：06 13 20 21 23 28 +16。\n说明：号码仅供娱乐参考。",
  "recommended_numbers": ["06", "13", "20", "21", "23", "28", "+16"],
  "llm_used": true,
  "generation_mode": "single_model",
  "candidates": [
    {
      "provider": "default",
      "model": "gpt-4o-mini",
      "reply": "推荐号码：06 13 20 21 23 28 +16。\n说明：号码仅供娱乐参考。",
      "recommended_numbers": ["06", "13", "20", "21", "23", "28", "+16"],
      "error": null
    }
  ],
  "decision_reason": null,
  "today_result": null
}
```

### 多模型响应示例

```json
{
  "session_id": "6505e80daea046fb88335404db8c9b4f",
  "game": "ssq",
  "reply": "综合多个模型的候选结果，推荐号码：03 09 12 18 24 31 +08。号码仅供娱乐参考。",
  "recommended_numbers": ["03", "09", "12", "18", "24", "31", "+08"],
  "llm_used": true,
  "generation_mode": "multi_model_discussion",
  "candidates": [
    {
      "provider": "model-a",
      "model": "gpt-4o-mini",
      "reply": "推荐号码：03 09 12 18 24 31 +08。",
      "recommended_numbers": ["03", "09", "12", "18", "24", "31", "+08"],
```

## 6. 历史号码兑奖校验

### 接口地址

```http
POST /api/v1/lottery/check
```

### 接口说明

前端把用户历史生成的号码和生成时间传给后端，后端查询对应日期的开奖数据，并判断是否中奖、中了几等奖、奖金多少。

默认逻辑：

- 不传 `draw_date`：使用 `generated_at` 的日期去查开奖数据。
- 传了 `draw_date`：使用指定开奖日期去查开奖数据。
- 浮动奖金如果开奖数据没有提供金额，返回 `prize_amount: null` 和 `amount_source: "unknown"`。
- 固定奖金奖级会使用内置规则返回金额，`amount_source: "built_in"`。

### 请求头

| 参数 | 必填 | 说明 |
|---|---|---|
| `Content-Type` | 是 | 固定为 `application/json` |
| `X-API-Key` | 条件必填 | 后端配置了 `API_ACCESS_KEY` 时必填 |

### 请求体

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|---|---|---|---|---|
| `game` | string | 否 | `ssq` | 彩票类型 |
| `numbers` | array[string] | 是 | 无 | 用户历史生成或购买的号码 |
| `generated_at` | string | 是 | 无 | 号码生成时间，ISO 格式，例如 `2026-04-30T10:30:00` |
| `draw_date` | string/null | 否 | null | 指定开奖日期，格式 `YYYY-MM-DD` |

### 请求示例

```json
{
  "game": "ssq",
  "numbers": ["01", "06", "11", "18", "22", "31", "+09"],
  "generated_at": "2026-04-30T10:30:00",
  "draw_date": "2026-04-30"
}
```

### 响应参数

| 字段 | 类型 | 说明 |
|---|---|---|
| `game` | string | 彩票类型 |
| `generated_at` | string | 号码生成时间 |
| `draw_date` | string | 实际用于比对的开奖日期 |
| `checked_numbers` | array[string] | 用户传入的号码 |
| `draw_numbers` | array[string] | 开奖号码 |
| `is_winning` | boolean | 是否中奖 |
| `prize_level` | string/null | 奖级，例如 `一等奖`、`六等奖`、`直选` |
| `prize_amount` | number/null | 奖金金额，单位元 |
| `currency` | string | 币种，固定为 `CNY` |
| `amount_source` | string | 金额来源：`draw_data`、`built_in`、`unknown`、`none` |
| `matched` | object | 命中详情 |
| `message` | string | 可直接展示给用户的结果说明 |
| `draw_result` | object/null | 开奖数据详情 |

### 响应示例

```json
{
  "game": "ssq",
  "generated_at": "2026-04-30T10:30:00",
  "draw_date": "2026-04-30",
  "checked_numbers": ["01", "06", "11", "18", "22", "31", "+09"],
  "draw_numbers": ["01", "06", "11", "18", "22", "31", "+09"],
  "is_winning": true,
  "prize_level": "一等奖",
  "prize_amount": 6800000,
  "currency": "CNY",
  "amount_source": "draw_data",
  "matched": {
    "red_count": 6,
    "blue_count": 1,
    "red_numbers": ["01", "06", "11", "18", "22", "31"],
    "blue_numbers": ["09"]
  },
  "message": "中奖，奖级：一等奖，奖金：6800000 元。",
  "draw_result": {
    "game": "ssq",
    "draw_date": "2026-04-30",
    "issue": "20260430",
    "numbers": ["01", "06", "11", "18", "22", "31", "+09"],
    "source": "local_file",
    "raw": {
      "draw_date": "2026-04-30",
      "issue": "20260430",
      "numbers": ["01", "06", "11", "18", "22", "31", "+09"],
      "prize_table": {
        "level_1": 6800000,
        "level_2": 180000,
        "三等奖": 3000,
        "四等奖": 200,
        "五等奖": 10,
        "六等奖": 5
      }
    }
  }
}
```

## 7. 开奖数据格式

开奖结果查询的数据源优先级：

1. `LOTTERY_API_URL`：你自己配置的开奖接口。
2. `LOTTERY_DATA_FILE`：本地 `data/lottery_results.json`。
3. 内置自动开奖源：`LOTTERY_AUTO_FETCH=true` 时启用。

相关环境变量：

```env
LOTTERY_API_URL=
LOTTERY_API_KEY=
LOTTERY_DATA_FILE=data/lottery_results.json
LOTTERY_AUTO_FETCH=true
LOTTERY_LATEST_IF_DATE_MISSING=true
```

说明：

- `LOTTERY_AUTO_FETCH=true`：前两种数据源都没有数据时，后端会尝试访问内置官方开奖源。
- `LOTTERY_LATEST_IF_DATE_MISSING=true`：如果指定日期没有开奖，会返回最近一期可用开奖结果，响应里的 `draw_date` 会显示实际开奖日期。
- 大模型不会作为开奖结果查询来源，因为大模型可能生成不准确的开奖结果。

如果使用本地文件 `data/lottery_results.json`，格式示例：

```json
{
  "ssq": [
    {
      "draw_date": "2026-04-30",
      "issue": "20260430",
      "numbers": ["01", "06", "11", "18", "22", "31", "+09"],
      "prize_table": {
        "level_1": 6800000,
        "level_2": 180000,
        "三等奖": 3000,
        "四等奖": 200,
        "五等奖": 10,
        "六等奖": 5
      }
    }
  ],
  "dlt": [],
  "fc3d": [],
  "pl3": [],
  "pl5": []
}
```

如果配置远程 `LOTTERY_API_URL`，服务会请求：

```http
GET {LOTTERY_API_URL}?game=ssq&date=YYYY-MM-DD
```

远程接口返回格式建议和上面的单条开奖记录一致。

## 8. 错误响应

### 密钥错误或未传

HTTP 状态码：

```text
401 Unauthorized
```

响应示例：

```json
{
  "detail": "Invalid or missing API key"
}
```

### 请求参数格式错误

HTTP 状态码：

```text
422 Unprocessable Entity
```

响应示例：

```json
{
  "detail": [
    {
      "type": "missing",
      "loc": ["body", "message"],
      "msg": "Field required"
    }
  ]
}
```
