# 彩票大模型 API 接口文档

## 1. 本次日志与配置变更

### 后端变更

- 不再使用 `.env`。
- 所有配置统一写在 `app/config.py`。
- Docker 启动命令改为 `python run.py`。
- `run.py` 会关闭 uvicorn 默认 access log。
- 请求日志由后端中间件统一打印，包含时间、IP、方法、路径、状态码、耗时。

日志示例：

```text
2026-05-03 14:22:01 182.150.57.90 GET /api/v1/lottery/today?game=ssq 200 35.20ms
```

### 需要前端配合

前端不需要改接口地址或请求体。

仍然按原来方式传请求头：

```http
X-API-Key: Tang
```

跨域默认已经放开：

```python
cors_allowed_origins = "*"
```

## 2. 配置文件

配置文件位置：

```text
app/config.py
```

常用配置：

```python
app_port = 8549
cors_allowed_origins = "*"
log_timezone = "Asia/Shanghai"
api_access_key = "Tang"

llm_base_url = "https://zenmux.ai/api/v1"
llm_api_key = os.getenv("LLM_API_KEY", "")
llm_model = "deepseek/deepseek-v4-pro-free"
```

修改 `app/config.py` 后，需要重新 build 镜像。

模型 key 不写进代码，启动容器时从系统环境变量注入：

```bash
docker run -d --name lottery-api \
  -p 8549:8549 \
  -e LLM_API_KEY="你的真实模型 key" \
  -v ./data:/app/data \
  lottery-api:latest
```

## 3. Docker 部署

构建镜像：

```bash
docker build -t lottery-api:latest .
```

停止旧容器：

```bash
docker stop lottery-api
docker rm lottery-api
```

启动新容器：

```bash
docker run -d --name lottery-api \
  -p 8549:8549 \
  -v ./data:/app/data \
  lottery-api:latest
```

查看日志：

```bash
docker logs -f --tail 100 lottery-api
```

## 4. 基础信息

服务地址：

```text
http://服务器IP:8549
```

鉴权方式：

```http
X-API-Key: Tang
```

支持的彩票类型：

| game | 名称 | 规则 |
|---|---|---|
| `ssq` | 双色球 | 6 个红球 01-33 + 1 个蓝球 01-16 |
| `dlt` | 大乐透 | 5 个前区 01-35 + 2 个后区 01-12 |
| `fc3d` | 福彩3D | 3 位数字，每位 0-9，可重复 |
| `pl3` | 排列三 | 3 位数字，每位 0-9，可重复 |
| `pl5` | 排列五 | 5 位数字，每位 0-9，可重复 |

## 6. 查询今日彩票开奖

```http
GET /api/v1/lottery/today?game=ssq
```

请求头：

| 参数 | 必填 | 说明 |
|---|---|---|
| `X-API-Key` | 是 | 后端配置的 `api_access_key` |

响应示例：

```json
{
  "game": "ssq",
  "draw_date": "2026-05-03",
  "issue": "20260503",
  "numbers": ["01", "06", "11", "18", "22", "31", "+09"],
  "source": "official_cwl_latest",
  "raw": {}
}
```

## 7. 聊天生成彩票号码

```http
POST /api/v1/chat/generate
```

请求体：

```json
{
  "session_id": "可选，不传则后端自动创建",
  "game": "dlt",
  "message": "给我一组大乐透号码",
  "include_today_result": true
}
```

响应重点字段：

| 字段 | 类型 | 说明 |
|---|---|---|
| `session_id` | string | 会话 ID |
| `recommended_numbers` | array[string] | 最终推荐号码 |
| `generation_mode` | string | `fallback`、`single_model`、`multi_model_discussion` |
| `candidates` | array[object] | 多模型候选结果 |
| `decision_reason` | string/null | 多模型最终选择理由 |

大乐透响应示例：

```json
{
  "game": "dlt",
  "recommended_numbers": ["02", "12", "19", "25", "30", "+03", "+09"]
}
```

前端展示：

```text
前区：02 12 19 25 30
后区：03 09
```

注意：不要只截取前 5 个号码。带 `+` 的两个号码是大乐透后区号码。

## 8. 历史号码兑奖校验

```http
POST /api/v1/lottery/check
```

请求体：

```json
{
  "game": "ssq",
  "numbers": ["01", "06", "11", "18", "22", "31", "+09"],
  "generated_at": "2026-05-03T10:30:00",
  "draw_date": "2026-05-03"
}
```

响应重点字段：

| 字段 | 类型 | 说明 |
|---|---|---|
| `is_winning` | boolean | 是否中奖 |
| `prize_level` | string/null | 奖级 |
| `prize_amount` | number/null | 奖金 |
| `matched` | object | 命中详情 |
| `message` | string | 可直接展示给用户 |

  "detail": "Invalid or missing API key"
}

```json
{
  "detail": [
    {
      "type": "missing",
      "loc": ["body", "message"],
      "msg": "Field required"
    }
  ]
}
```
